Software Requirements Specification (SRS) Document: Employee Recruitment System
1. Introduction
   1.1 Purpose
   The purpose of this document is to provide a detailed description of the requirements 
   for the Employee Recruitment System. It outlines the functional and non-functional requirements.
  

   1.2 Scope
   The Employee Recruitment System is a web-based application that allows organizations to 
   post their staffing requirements and view profiles of interested candidates
   The system will provide has facilites where prospective candidates can upload their CV and 
   apply for jobs suited to them.
   The primary objective to develop this system to optimize the recruitment process for an Organisation


   Functional Requirements
   
	  -	Each candidate can visit the Employee Recruitment System 
	  
      - Administrators should be able to add, update, and delete Candidate records.
	  
      - Candidate record should include details such as id,Name,Address,Adhar_Card,Photo,date,job_id
	  
      - The system should store and display Candidate photos, if available.
	  
	  - Send email to the registered candidates by registerid and Name of the candidate
	  
	  - Candidate can login using email and password
	  
	  - candidate can change address only after registered
